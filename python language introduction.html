الـ Generators و Yield

لتخطي مشاكل نفاذ الذاكرة (Memory Crash) عند معالجة ملفات بيانات ضخمة بالملايين، يفضل توليد القيم قيمة تلو الأخرى بدلاً من حفظها دفعة واحدة.

السر المحترف: 
def get_records():
    for row in big_file:
        yield row # لا يحجز ذاكرة!

طريقة معالجة البيانات بالحلقات التقليدية

استخدام الحلقات التكرارية (for loop) للمرور على مجموعات هائلة من الأرقام في NumPy أو Pandas يقتل سرعة المعالجة تماماً ويصنف كأحد الأخطاء الكارثية.

Python 3 
# الطريقة الخاطئة والبطيئة جداً
output_list = []
for item in dataset:
    if item > 50:
        output_list.append(item * 2)
# هذه العملية تأخذ ثوانٍ ثمينة من وقت المعالجة!

# الطريقة الذكية والصحيحة (بايثون الاحترافية)
import numpy as np

# العمليات الحسابية الموجهة سريعة في لحظة واحدة!

الأخطاء الكارثية الشائعة وحلولها البرمجية

أكورديون تفاعلي يعرض المشاكل الشائعة التي يقع فيها مبرمجو البيانات وطريقة تفاديها

المشكلة: عند تمرير قائمة فارغة كقيمة افتراضية لمعامل دالة def add_item(item, items_list=[])، يتم إنشاء هذه القائمة مرة واحدة فقط وتظل مخزنة في الذاكرة لتراكم كل المدخلات القادمة في المرات القادمة بدلاً من إنشاء قائمة فارغة جديدة في كل استدعاء!

❌ الخطأ: 
def append_to(element, target=[]):
    target.append(element)
    return target
# سيتراكم بشكل خاطئ في الذاكرة!
✅ الحل الصحيح: 
def append_to(element, target=None):
    if target is None:
        target = []
    target.append(element)
    return target

المشكلة: تظهر الرسالة الشهيرة SettingWithCopyWarning عند محاولة تعديل قيمة في DataFrame تم استخلاصها كنسخة جزئية وليس كمرجع أساسي، مما يؤدي لعدم معرفة بايثون ما إذا كان التعديل يتم في نسخة وهمية أم في البيانات الأصلية.

❌ الخطأ: 
df[df['age'] > 30]['status'] = 'Senior'
# تسبب تحذيراً ولا تغير في الأصل غالباً
✅ الحل الصحيح: 
# استخدام الدالة المخصصة .loc للوصول الآمن
df.loc[df['age'] > 30, 'status'] = 'Senior'

المشكلة: تجميع وتخزين خسارة التدريب (Loss Value) في PyTorch ككائن tensor متكامل داخل قائمة، مما يجعل بايثون تحتفظ بكامل شجرة العمليات الحسابية والرسومية في كارت الشاشة والذاكرة دون تصفيتها، مما يسبب تحطم البرنامج بعد تكرار الدورات (Epochs).

❌ الخطأ: 
total_loss += loss # خطأ قاتل!
# يحتفظ بكل المسار الحسابي للـ GPU
✅ الحل الصحيح: 
# استخلاص القيمة الرقمية النظيفة فقط
total_loss += loss.item()

1. تعديل القوائم الافتراضية كمعاملات للدوال

تجنب تمرير قائمة فارغة كمعامل افتراضي في الدوال، واستخدم معامل None بدلاً منها لتفادي تراكم البيانات المخزنة عشوائياً في الذاكرة.

2. خطأ الإسناد المتسلسل في Pandas

استخدم دائماً الدالة loc للوصول والتعديل الآمن في حقول البيانات وتجنب Chained Assignment لتفادي Warning الأداء الشهير.

3. مشكلة تسرب الذاكرة في الذكاء الاصطناعي

عند جمع الـ loss في PyTorch، استخدم دالة item لاستخلاص القيمة وتجنب تخزين التينسور بالكامل لحماية كرت الشاشة والذاكرة من النفاذ.
